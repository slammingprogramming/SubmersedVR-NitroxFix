# Contributing

Thanks for helping improve SubmersedVR-NitroxFix. Contributions can include compatibility reports, documentation fixes, reproducible bug reports, troubleshooting notes, or source changes.

## Good First Contributions

- Submit a [compatibility report](.github/ISSUE_TEMPLATE/compatibility_report.md) for a headset, controller, runtime, connection method, or storefront.
- Improve [Troubleshooting](docs/TROUBLESHOOTING.md) with a verified workaround.
- Update [Known Issues](docs/KNOWN_ISSUES.md) with clearer symptoms or reproduction steps.
- Clarify [Installation](docs/INSTALLATION.md) for a setup path that confused you.

## Before Opening An Issue

Check these pages first:

- [Known Issues](docs/KNOWN_ISSUES.md)
- [Troubleshooting](docs/TROUBLESHOOTING.md)
- [Compatibility](docs/COMPATIBILITY.md)
- [FAQ](docs/FAQ.md)

Use the matching issue template so maintainers receive the right details:

- [Bug report](.github/ISSUE_TEMPLATE/bug_report.md)
- [Compatibility report](.github/ISSUE_TEMPLATE/compatibility_report.md)
- [Documentation issue](.github/ISSUE_TEMPLATE/documentation.md)
- [Question](.github/ISSUE_TEMPLATE/question.md)

## Bug Reports

Useful bug reports include:

- Clear reproduction steps.
- Expected and actual behavior.
- `BepInEx/LogOutput.log`.
- Subnautica, Nitrox, SubmersedVR, SubmersedVR-NitroxFix, BepInEx, and SteamVR versions.
- Headset, controller, runtime, connection method, and storefront.
- Host/client role and server type.

## Compatibility Reports

Compatibility reports should include the full setup, whether multiplayer worked, whether motion controls worked, and any special setup steps. These reports help keep [Compatibility](docs/COMPATIBILITY.md) accurate.

## Pull Requests

When opening a pull request:

1. Keep the change focused.
2. Link related issues or reports.
3. Update docs when behavior or support guidance changes.
4. Include screenshots, logs, or reproduction notes when they help reviewers.

## Documentation Style

- Prefer short, direct instructions.
- Use relative links between repository files.
- Put file paths, launch arguments, and config keys in backticks or fenced `text` blocks.
- Keep compatibility statuses aligned with the legend in [Compatibility](docs/COMPATIBILITY.md#legend).
